# نظام إدارة الجودة والمشاريع (SOP System)

نظام داخلي كامل (Laravel + Vue.js) يُطبّق دليل إجراءات العمل الموحّد (SOP) لبناء مواقع WordPress على إطار Astra. يتيح للمبرمجين تسجيل إنجاز كل بند من بنود الجودة أثناء العمل، ولمراجع الجودة (QA) تنفيذ مراجعة نهائية شاملة قبل اعتماد المشروع وتسليمه.

## المكوّنات

- **backend/** — Laravel 13 API (Sanctum، SQLite افتراضيًا)
- **frontend/** — Vue 3 + Vite + Pinia + Vue Router + Tailwind CSS (واجهة عربية RTL بالكامل)

## أهم المزايا

- **سير عمل بـ 5 مراحل** (التحليل البصري → تجهيز الأساس → نظام التصميم → بناء الصفحات → الضبط والتسليم) يُنشأ تلقائياً مع كل مشروع.
- **قائمة تحقق كاملة (71 بنداً موزّعة على 15 قسماً)** مأخوذة حرفياً من الوثيقة: الهوية البصرية، المكوّنات، الحركة، الأداء، التوافق، SEO/Screaming Frog، الصور، الترجمة وRTL، الـ Responsive، وقائمة الإطلاق. كل بند يدعم: تحديد كـ "منجز/غير منطبق"، ملاحظات، ورفع أدلة (ملفات أو روابط).
- **حوكمة الإضافات**: طلب/موافقة/رفض أي إضافة جديدة من قِبل المدير أو أخصائي تقنية المعلومات.
- **إدارة التراخيص** (Elementor Pro / Astra Pro) مع تتبع تاريخ الانتهاء والمسؤول عن التجديد.
- **تقارير الأداء** (Lighthouse / PageSpeed / Screaming Frog) بمقارنة القالب الأصلي مقابل الموقع النهائي، مع **فحص تلقائي حقيقي** عبر Google PageSpeed Insights API (نتيجة موبايل/ديسكتوب + قائمة مفصّلة بالمشاكل المكتشفة وسبب كل مشكلة). مفتاح الـ API يُدار من صفحة "الإعدادات" ويتفعّل فوراً دون إعادة نشر.
- **مراجعة جودة نهائية (QA)**: مراجع الجودة يفحص كل بند تم إنجازه (Pass/Fail + ملاحظة) قبل اتخاذ قرار الاعتماد أو طلب تعديلات.
- **جدول اعتماد رقمي** مطابق لجدول التوقيعات في الوثيقة الأصلية (إعداد / مراجعة / اعتماد نهائي)، مع تسجيل تلقائي لكل توقيع وتاريخه.
- **لوحة تحكم** بمؤشرات النجاح المذكورة في القسم 8 من الوثيقة (متوسط Lighthouse، متوسط عدد الإضافات، حالة المشاريع، سجل النشاطات).
- **أدوار وصلاحيات مرنة**: 4 أدوار أساسية (مدير/CTO، مبرمج، مراجع جودة، أخصائي تقنية معلومات) + إمكانية إنشاء أدوار مخصصة (مثل "المدير التنفيذي") وتحديد صلاحياتها بدقة من صفحة "الأدوار والصلاحيات" دون الحاجة لتعديل الكود.
- **تصدير تقرير PDF احترافي لكل مشروع**: ملخص قائمة التحقق بالنسب المئوية لكل قسم، مقارنة الأداء (قبل/بعد)، وجدول الاعتماد الرقمي — أداة عرض جاهزة للعميل.
- **بحث وفلترة على صفحة المشاريع** (بالاسم/العميل، الحالة، والمبرمج المكلّف).
- **سجل نشاطات كامل** بصفحة مستقلة مع فلترة حسب المشروع ونوع الحدث، وليس فقط آخر 15 نشاطاً بلوحة التحكم.
- **استيراد تقرير Screaming Frog**: رفع تصدير CSV من Screaming Frog (Internal > All — يشمل HTML وCSS وJS والصور والخطوط، أو Internal > HTML لفحص أسرع) ليفحص النظام تلقائياً عدد H1، وسم noindex، Canonical الذاتي، الروابط المكسورة (على كل أنواع الملفات)، التحويلات المؤقتة، طول Title/Meta Description وتكرارها، المحتوى المكرر، أحجام الصور، وعدد الإضافات المكتشفة من روابط الأصول — ويحدّث بنود قائمة التحقق المطابقة (الأقسام 3 و4 و5) تلقائياً بدل التعليم اليدوي.
- **فحص اكتمال الترجمة (اختياري)**: مع تفعيل Custom Extraction بسيط في Screaming Frog، يفحص النظام كل صفحة عربي بحثاً عن نص إنكليزي متبقٍ غير مترجم أو مصطلحات شائعة مترجمة خطأ (مثل "الأمنيات" بدل "المفضلة")، دون الادّعاء بفحص جودة الترجمة نفسها.

## التشغيل محلياً

### المتطلبات
PHP 8.2+, Composer, Node 18+, npm

### الخلفية (Backend)

```bash
cd backend
composer install
cp .env.example .env    # إذا لم يكن .env موجوداً
php artisan key:generate
touch database/database.sqlite
php artisan migrate --seed
php artisan storage:link
php artisan serve        # يعمل على http://localhost:8000
```

### الواجهة (Frontend)

```bash
cd frontend
npm install
cp .env.example .env    # أو أنشئ ملف .env يحوي VITE_API_URL
npm run dev              # يعمل على http://localhost:5173
```

تأكد أن `VITE_API_URL` في `frontend/.env` يشير إلى رابط الـ API الخلفي (مثلاً `http://localhost:8000/api`).

## دورة حياة المشروع في النظام

1. المدير ينشئ مشروعاً جديداً ويحدد المبرمجين المكلّفين — يتم تلقائياً إنشاء المراحل الخمس وقائمة التحقق الكاملة (71 بنداً) وتوقيع "الإعداد".
2. المبرمجون يعملون على المشروع، ويحددون كل بند "منجز" فور الانتهاء منه مرفقاً بدليل (صورة/رابط) عند الحاجة، ويحرّكون حالة كل مرحلة.
3. أي إضافة WordPress جديدة تتطلب طلباً صريحاً وموافقة المدير أو أخصائي تقنية المعلومات.
4. عند الانتهاء، مراجع الجودة يبدأ "مراجعة جودة" تفحص كل بند من الـ 71 بنداً (Pass/Fail)، ثم يصدر قراره: اعتماد / طلب تعديلات / رفض.
5. عند الاعتماد، يقوم المدير بالاعتماد النهائي وتسليم المشروع، ويُسجَّل ذلك في جدول التوقيعات (إعداد / مراجعة / اعتماد نهائي).
